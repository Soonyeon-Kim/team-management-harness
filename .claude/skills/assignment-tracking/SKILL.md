# 프로젝트 투입 이력 워크플로우

컨설팅 조직에서 프로젝트 투입은 관리의 중심축이다. 투입 이력의 목적은 단순 기록이 아니라 **가동률(과투입/유휴) 파악, 휴가·프로젝트 일정 충돌 사전 감지, 역량 매칭·갭 분석의 근거 제공**이다. 이 데이터는 leave-manager(휴가 충돌)와 growth-coach(역량 매칭/갭)가 함께 참조한다.

> 데이터 위치·스키마는 `team-data-store` 스킬을 따른다. 투입 이력은 `team-data/assignments/assignments.json`.

## 작업 유형

### 1. 투입 등록
1. `assignments.json`을 읽고 새 ID(`A-{최대값+1}`)를 부여한다. `member`가 `members/`에 존재하는지 확인.
2. 필드(member, project, client, role, allocation, start, end, status, notes)를 채운다. 진행 중이면 `end: null`, `status: active`. 예정이면 `planned`.
3. 등록 시 **가동률 점검**: 해당 팀원의 active 투입 allocation 합이 100%를 넘으면 과투입으로 보고한다.

### 2. 철수 / 종료
1. 대상 투입을 ID 또는 프로젝트명으로 찾는다.
2. read-modify-write로 `end`를 채우고 `status`를 `ended`로 바꾼다. 다른 투입은 건드리지 않는다.
3. 종료 시 leave-manager에 "프로젝트 종료 후 휴가 계획" 수립을, growth-coach에 "프로젝트 중 발견된 역량 갭" 정리를 제안한다(해당 도메인 파일은 직접 쓰지 않는다).

### 3. 투입 현황 / 가동률 리포트
- 팀원별 active 투입과 allocation 합(가동률)을 정리한다. **과투입(>100%)**과 **유휴(active 0%)**를 양방향으로 플래그한다.
- 진행 중 프로젝트의 종료 임박(`end`가 다가오는 active 건)을 보고한다.

### 4. 휴가·역량 교차 지원 (팬아웃용)
- **휴가 충돌**: leave.json의 휴가 기간이 active 투입 기간과 겹치면, 프로젝트 일정 충돌로 leave-manager에 신호를 보낸다.
- **역량 매칭**: 신규 투입 검토 시 growth-coach의 TO-BE 목표 역량을 쓸 수 있는 프로젝트인지 매칭 의견을 제시한다.

## 핵심 규칙

- **가동률 우선 보고**: 현황 나열보다 과투입·유휴·종료 임박을 먼저 띄운다.
- **member 무결성**: 존재하지 않는 팀원의 투입을 만들지 마라.
- **삭제 금지**: 종료된 투입은 `ended`로 보존한다(역량 매칭·이력 추적 근거).
- **경계 존중**: 휴가 처리는 leave-manager, 역량 기록은 growth-coach, 자격증은 certification-manager 영역이다. 충돌·갭을 감지해 넘기되 그 도메인 파일을 직접 쓰지 마라. 이 스킬은 투입 이력 자체만 소유한다.
- **추정 금지**: allocation·종료일을 임의로 만들지 마라. 팀장이 제공한 값만 기록한다.

## 이전 산출물이 있을 때
- "투입 현황 다시" 요청이면 assignments.json을 재조회해 최신 가동률로 리포트한다(캐시 재사용 금지).
