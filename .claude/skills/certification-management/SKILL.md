# 자격증 관리 워크플로우

자격증 관리의 목적은 단순 목록 보관이 아니라 **마감을 놓치지 않고(신청·시험), 취득을 업무 가치로 연결하며(연관성·유지보수), 합격을 보상으로 마무리(환급)**하는 것이다. 마감 임박과 유지보수 만료를 항상 함께 띄운다.

> 데이터 위치·스키마는 `team-data-store` 스킬을 따른다. 자격증은 `team-data/certifications/certs.json`, 팀 목표는 `team-data/certifications/team-goals.md`.

## 월간 체크 작업

### 1. 팀원별 보유 자격증 현황 파악
- `certs.json`을 팀원별로 묶어 status(target/in-progress/acquired/expired)별로 정리한다.
- `members/`의 slug와 대조해 무결성을 확인한다.

### 2. 취득 계획 수립
- 팀원의 역할·프로젝트(members 프로필, growth 데이터가 주어지면 참조)와 **업무 연관성**이 높은 자격증을 target으로 등록한다. `relevance`에 왜 필요한지 적는다.
- 막연한 "따면 좋은" 자격증이 아니라 업무·비즈니스에 연결된 것만 권한다.

### 3. 시험 일정 / 신청 마감 확인
다음을 위험도 순으로 보고한다:
- **신청 마감 임박**: `application_deadline`이 오늘~14일 이내이고 미신청.
- **시험 임박**: `exam_date`가 오늘~14일 이내.
- 마감 지난 미신청 건은 별도 플래그.

### 4. 합격 환급 처리
- status를 `acquired`로 바꾸고 `acquired_date`를 기록한다.
- `reimbursed`/`reimburse_amount`로 환급 처리 상태를 남긴다. 미처리면 "환급 신청 필요"로 보고한다.
- 합격 시 **축하 메시지 초안**을 함께 제시한다(팀장이 전달하도록).

## 분기 체크 작업

- **유지보수 점검**: `status: acquired`이면서 `maintenance_due` 또는 `expiry_date`가 다가오는 자격증을 보고(Salesforce 등은 주기적 유지보수 필요). 놓치면 자격이 만료된다.
- **팀 차원 목표**: `team-goals.md`에 분기 팀 자격증 목표를 정리/갱신한다.
- **파트너십 혜택**: 벤더 파트너십(무료 응시권·교육 등) 활용 방안을 `team-goals.md`에 정리하고 적용 가능한 팀원을 매칭한다.
- **교육 세션 독려**: 취득 계획(target) 대비 교육 참여가 저조한 팀원을 식별한다.

## 핵심 규칙

- **마감 우선 보고**: 신청 마감·유지보수 만료는 놓치면 복구 불가다. 현황 나열보다 마감 임박 건을 맨 위에 둔다.
- **업무 연관성 필수**: target 등록 시 `relevance`를 비우지 마라. 연관성 없는 자격증은 권하지 않는다.
- **환급은 사실 기반**: 합격·환급 처리는 팀장이 확인한 사실로만 기록한다. 임의로 acquired/reimbursed 처리하지 마라.
- **member 무결성**: 존재하지 않는 팀원의 자격증을 만들지 마라.

## 이전 산출물이 있을 때
- "자격증 현황 다시" 요청이면 certs.json을 재조회한다. 만료/취소된 자격증도 삭제하지 않고 status로만 표시한다(이력 보존).
