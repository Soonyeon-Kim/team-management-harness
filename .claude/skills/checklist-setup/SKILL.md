# 팀장 체크리스트 셋업 워크플로우

팀장 체크리스트는 이 하네스의 척추다 — 4개 관리 영역(1:1, 업무, 성장, 휴가)을 하나의 정기 루틴으로 묶는다. 체크리스트가 곧 "주간 점검 때 무엇을 보는가"를 정의하고, 오케스트레이터는 이를 그대로 실행한다. 따라서 추상적 다짐("팀원을 잘 챙기자")이 아니라 **담당 도메인이 매핑된 실행 가능한 점검 항목**으로 만든다.

> 데이터 위치·스키마는 `team-data-store` 스킬을 따른다. 결과물은 `team-data/checklist.md`.

## 워크플로우

### 1. 저장소 초기화 (최초 실행 시)
- `team-data/`가 없으면 디렉토리 구조와 빈 `tasks.json`(`{"tasks": []}`), `leave.json`(`{"leave": []}`)을 만든다.
- `team.md`를 팀명·팀장·케이던스(기본 weekly)로 생성한다.

### 2. 팀원 명단 등록
- 팀원이 없으면 팀장에게 명단(이름·역할·입사일)을 물어 `members/{slug}.md`를 생성한다.
- slug는 영문 kebab-case로 부여하고 팀장에게 확인받는다.

### 3. 체크리스트 함께 정의
팀장과 대화로 항목을 도출한다. **각 항목은 반드시 담당 도메인(에이전트)에 매핑**하고 **cadence(주간/월간/분기/특별)**를 정한다. 이 팀의 체크리스트는 3대 축이다 — 아래를 출발점으로 제시하되 팀장 상황에 맞게 가감한다:

- **① 자격증 관리** (certification-manager): 월간(보유 현황·취득 계획·시험/신청 마감·환급), 분기(유지보수·팀 목표·교육 독려·파트너십 혜택)
- **② 휴가 현황 관리** (leave-manager): 주간(연차 현황·향후 2주·프로젝트 충돌·미사용 독려), 월간(누적률 100~150%·마이너스 연차·밸런스), 특별(신입 3/31 소멸·프로젝트 종료 후 계획·휴가비 100만원)
- **③ 역량 향상 AS-IS⇒TO-BE** (growth-coach + oneonone-coach): AS-IS(현재 역량·갭), TO-BE(6개월/1년 목표·지표), 실행·모니터링(월간 1:1·교육·멘토링·역량 매칭), 분기 평가

`task-tracker`(업무/태스크)는 체크리스트 외 보조 도구다. 팀장이 원하면 별도 cadence로 추가할 수 있으나 기본 3축에는 넣지 않는다.

팀장에게 "더할 항목, 뺄 항목"을 명시적으로 묻는다.

### 4. checklist.md 작성
cadence별 섹션(`## 주간`/`## 월간`/`## 분기`/`## 특별 관리`)으로 나누고, 각 항목 끝에 `(담당: {에이전트})`를 붙인다. frontmatter에 `version`, `updated`를 둔다. 실제 작성 예시는 이 팀의 `team-data/checklist.md`를 참조한다.

## 핵심 규칙

- **항목은 담당 매핑 필수**: 담당 에이전트가 없는 항목은 만들지 마라. 오케스트레이터가 실행할 수 없다.
- **검증 가능하게**: "팀원을 챙긴다"(X) → "마지막 1:1로부터 2주 넘은 팀원이 0명"(O).
- **팀장 주도**: 항목을 임의로 확정하지 말고 반드시 팀장의 가감을 받는다. 이 체크리스트는 팀장의 관리 철학을 담는다.
- **버전 관리**: 항목 변경 시 `version`을 올리고 `updated`를 갱신한다. 기존 항목 삭제도 팀장 승인 후에만.

## 이전 산출물이 있을 때

- `checklist.md`가 이미 있으면 "항목 추가/수정"으로 간주하고 기존 내용을 읽어 보완한다. 전체를 새로 쓰지 마라.
