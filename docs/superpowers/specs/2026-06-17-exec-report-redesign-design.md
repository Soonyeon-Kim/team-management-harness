# 상급자 보고용 HTML 리포트 — UI/UX 디자인 개선 설계

- **날짜:** 2026-06-17
- **대상:** `exec-report-web` 스킬 (`.claude/skills/exec-report-web/template.html` + `SKILL.md`)
- **범위:** 시각 + 레이아웃 다듬기 (정보구조·KPI 산식은 유지)
- **상태:** 디자인 확정. 본 문서는 구현 전 설계 스펙이다.

## 1. 목표

상급자에게 보고하는 자체완결 HTML 리포트의 **시각적 완성도와 가독성**을 끌어올린다. 산식·데이터 소스·정보구조(헤더 → KPI → 위험 → 팀원표 → 영역요약 → 푸터)는 그대로 두고, **모던 SaaS(Refactoring UI 계열)** 결로 모양·색·여백·깊이감만 재정의한다.

### 디자인 방향 결정 (브레인스토밍 합의)

| 결정 항목 | 선택 | 비고 |
|---|---|---|
| 전체 결 | **모던 SaaS** | Stripe·Linear류. 옅은 그림자 깊이, 액센트 1색 절제, 넉넉한 여백 |
| 헤더 | **B · 라이트 카드 + 액센트** | 흰 카드 + 좌측 5px 액센트 바 + 키커 라벨. 다크 헤더(현행) 대체 |
| KPI 카드 | **B · 상태색 배경 + 컬러 숫자** | ok/warn/bad 카드 배경 옅게 + 숫자에도 상태색. "5초 스캔" 강화 |
| 위험 신호 | **A · 좌측 심각도 바 + CSS 색 점 + 배지** | 이모지(🔴) 대신 **순수 CSS 색 점**. 색 최소화로 KPI와 충돌 회피 |
| 팀원 표 | **A · 정돈된 표** | 헤어라인 행 + 둥근 막대 + 호버 하이라이트. 인원 확장에 유리 |
| 영역 요약 | 다듬은 카드 | 아이콘 칩 헤더 + CSS 점 불릿. 5번째(1:1) 카드는 가로 전체 |
| 액센트 색 | **인디고 `#4f46e5`** | 헤더 바·섹션 마커·막대 기본색 |

## 2. 디자인 토큰 (구현 시 `:root`에 반영)

```css
--ink:#0f172a; --muted:#64748b; --line:#e7ebf0; --accent:#4f46e5;
--ok:#16a34a; --warn:#d97706; --bad:#dc2626; --info:#2563eb;
--ok-bg:#f0fdf4; --warn-bg:#fffbeb; --bad-bg:#fef2f2; --info-bg:#eff6ff;
/* 페이지 배경(스테이지): #eef1f6 / 카드: #fff */
```

- **폰트:** `"Segoe UI","Malgun Gothic","맑은 고딕",-apple-system,Roboto,sans-serif`
  — 한글 렌더를 위해 Malgun Gothic을 스택에 유지(웹폰트 금지 제약 준수).
- **그림자(깊이):** `0 1px 2px rgba(15,23,42,.05), 0 4px 12px rgba(15,23,42,.05)`
- **모서리:** 카드 14px · 내부 요소(막대/행) 10~12px · 배지/점 999px
- **간격 체계:** 8의 배수 기준(12/14/16/22) · 섹션 간 `margin-top:22px`
- **타이포:** h1 22px / KPI 값 27px·weight 760 / 섹션 헤더 14px / 본문 12.5~14px / 라벨 11px uppercase·letter-spacing .03em

## 3. 컴포넌트별 사양

### 3.1 헤더 (라이트 카드 B)
- 흰 카드, `::before`로 좌측 5px 액센트 세로 바, 옅은 그림자.
- 좌: 키커 라벨 `팀 현황 보고`(액센트색·uppercase) → h1 `{{TEAM_NAME}}` → 서브 `팀장 {{TEAM_LEAD}} · 보고 기준일 {{REPORT_DATE}}`.
- 우: `🔒 내부 보고용` 배지(중립 회색 pill).
- **변경점:** 현행은 h1이 `{{TEAM_NAME}} 팀 현황 보고`였음 → **키커(고정 문구) + h1=팀명**으로 분리. 다크 배경 → 라이트.

### 3.2 핵심 지표 (KPI B)
- 5칸 그리드. 카드 배경/테두리를 상태(`ok`/`warn`/`bad`)로 옅게, **값 숫자에도 상태색**.
- 중립 KPI(팀 인원·자격증)는 흰 배경·검정 숫자.
- 구조: `.l`(라벨) / `.v`(값) / `.n`(노트).
- 상태 클래스·산식·휴가비 규칙은 현행 `SKILL.md`/`team-data-store` 그대로(재정의 없음).

### 3.3 핵심 위험 신호 (A + CSS 점)
- 흰 카드 행 + 좌측 4px 심각도 바(`hi`=bad / `mi`=warn / `lo`=info).
- 아이콘: **순수 CSS 원(10px) + 은은한 후광 링**(`box-shadow 0 0 0 4px rgba(...,.14)`). 이모지 미사용.
- 우측 심각도 배지(pill): `즉시 조치`(hi) / `주의`(mi) / `참고`(lo).
- 구조: `.ico>.dot` / `.t`(제목) / `.d`(상세) / `.pill`.

### 3.4 팀원 현황 (표 A)
- 흰 표, 헤어라인 행 구분(`#f1f5f9`), 헤더행 `#f8fafc`·uppercase 라벨, 행 호버 `#fafbff`.
- 가동률·연차 누적률은 **둥근 막대**(`height:7px`, `border-radius:999px`) + 막대 아래 라벨.
- 막대색: `ok/warn/bad`, 기본 액센트.
- 컬럼/토큰은 현행 유지(`M_NAME`·`M_ROLE`·`M_UTIL_*`·`M_LEAVE_*`·`M_CERT`·`M_LASTONE`·`M_GROWTH`).

### 3.5 영역별 요약 (다듬은 카드)
- 2열 그리드. 카드마다 아이콘 칩(`26x26`, `#eef2ff` 배경) + 제목.
- 리스트는 `list-style:none` + **CSS 점 마커**(`li::before` 5px 액센트 원). 이모지 불릿 제거.
- 5번째 카드(1:1)는 `grid-column:1/-1`로 가로 전체 → 홀수 카드 균형.
- 5개 영역·`{{AREA_*_LIST}}` 토큰은 현행 유지.

### 3.6 푸터
- 헤어라인 상단 보더 + 좌우 정렬(생성 일시·출처 / 툴명). `{{GENERATED_AT}}` 유지.

## 4. 치환 계약(토큰) 변경 요약

대부분의 `{{TOKEN}}`·`REPEAT` 계약은 **유지**된다. 구현 시 `SKILL.md`를 함께 갱신해야 하는 항목만 정리:

| 항목 | 현행 | 변경 |
|---|---|---|
| 헤더 제목 | h1 = `{{TEAM_NAME}} 팀 현황 보고` | 키커(고정 "팀 현황 보고") + h1 = `{{TEAM_NAME}}` |
| 위험 아이콘 | `{{RISK_ICON}}` (🔴/🟠/🔵) | **제거** → 심각도 클래스로 CSS 점 자동 렌더 |
| 위험 배지 | (없음) | **신규 `{{RISK_PILL}}`** — sev에서 파생(hi=즉시 조치/mi=주의/lo=참고) |
| 그 외 토큰 | — | 변경 없음(KPI·member·area·footer 토큰 동일) |

> KPI 값에 상태색을 입히는 것은 기존 `{{KPI_*_STATUS}}` 클래스로 CSS에서 처리되므로 **신규 토큰 불필요**.

## 5. 제약 (반드시 준수)

- **자체완결:** 외부 CDN·웹폰트·스크립트·이미지 URL 금지. 모든 스타일 인라인 `<style>`, 차트는 CSS 막대만.
- **인쇄/PDF(`@media print`):** KPI 상태 배경·위험 색·막대 색을 `print-color-adjust:exact`로 보존. `break-inside:avoid` 유지. 헤더가 **라이트**로 바뀌므로, 현행의 "헤더 다크 배경 강제" 인쇄 규칙은 제거/수정.
- **반응형:** KPI 5→2열, 영역 2→1열, 헤더 세로 스택(좁은 화면).
- **빈/단일 데이터 견고성:** 팀원 1명·투입 전부 종료·1:1 없음에서도 깨지지 않음. `0%`/`null`은 `0`/`—`. "데이터 미입력"을 위험으로 과장 금지.

## 6. 범위 밖 (Non-goals)

- KPI 산식·임계값·휴가비 규칙·가동률 정의 변경 없음(`team-data-store`/도메인 스킬 소관).
- 정보구조(섹션 종류·순서) 변경 없음.
- 새 데이터·새 에이전트·실행 코드(JS) 추가 없음.

1. `template.html`의 `<style>`·마크업을 본 사양으로 교체(토큰/REPEAT 마커는 보존).

4. 테스트 시나리오 재확인: (a) 정상 다인원, (b) 엣지(1명·투입 종료·1:1 없음) 모두 레이아웃 정상·색 보존.
