## AS-IS (현재 상태)
- 기술 스택 / 역량 수준: Python·Docker·Kubernetes·CI/CD·모델 서빙 상급. 데이터 파이프라인·MLOps는 팀 내 최강. 통계·실험 설계는 중하.
- 역량 갭 (프로젝트 중 발견): 이커머스 추천 프로젝트에서 모델 후보 선정 근거를 스스로 세우지 못해 데이터 사이언티스트 의존. 모델링 이론 부족이 드러남.
- 강점 / 개선 영역: 강점은 운영·자동화·장애 대응. 개선은 ML 이론과 동시 다중 프로젝트 우선순위 관리.
- 업무 만족도 / 성장 욕구: 만족도 보통. 과투입으로 피로 누적. "엔지니어 + 모델러" 하이브리드로 성장하고 싶어 함.
## TO-BE (목표)
- 6개월 목표: AWS Machine Learning Specialty 취득. 추천 모델의 후보 모델을 직접 비교·선정할 수 있는 수준 도달.
- 1년 목표: MLOps 리드로서 팀 추론 파이프라인 표준(모니터링·롤백 포함)을 설계·문서화.
- 비즈니스 방향성 연계: 모델 운영 안정화는 고객 재계약율과 직결 — 운영 SLA 책임 영역 확대.
- 측정 가능한 성과 지표: 자격증 1건 취득, 추론 파이프라인 표준 문서 1건, 동시 투입 ≤100%로 조정.
- 확보 리소스 / 예산: AWS 자격증 응시료·교재 환급, 사내 ML 이론 스터디 참여.
## 실행 & 모니터링
- 월간 1:1 진척 (oneonone-coach 연동): 과투입 부하와 자격증 학습 진척을 매월 점검.
- 교육 프로그램 참여: 사내 통계/실험설계 스터디 격주 참여 중.
- 멘토링 / 코칭: 윤소희와 상호 멘토링 — 강하늘은 엔지니어링, 윤소희는 모델링 전수.
- 프로젝트 역량 매칭: 과투입 해소 위해 종료 예정 프로젝트 철수 후 단일 프로젝트 집중 배치.
## 성과 평가 (분기)
- 분기 역량 향상도: (2분기) 자격증 학습 순항, 실험 설계 이해 향상 중.
- 성공 사례: 통신사 모델 서빙 무중단 배포 파이프라인 구축으로 운영 안정화.
- 추가 지원 방안: 과투입 즉시 해소 — 부하 분산이 최우선 지원.
- 차기 목표 수정: 자격증 취득 후 모델링 실무 과제 1건 단독 부여.
